KEY DESIGN DECISIONS (so a fresh session doesn't second-guess)
--------------------------------------------------------------
- Statement::InlineAsm carries `address: u64` deliberately; losing it would be
  expensive for later passes to rebuild. Address is the anchor later structuring
  passes will use to merge/replace AST nodes.
- Lifter produces return_type = TypeInfo::Void, parameters = [], is_variadic = false.
  Signatures are reconstructed later once calling-convention recovery lands.
- Lifter is "minimum honest": it preserves order + addresses, emits no structure,
  no type recovery. Every later pass strictly adds information.
- Optimizer's match on Statement keeps `_ => {}` catch-alls, so adding InlineAsm
  did not break it. Constant folding never touches InlineAsm (correct — the
  payload is a string, not an expression).
